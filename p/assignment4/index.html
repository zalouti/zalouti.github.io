Part Two: Matchmaker

Milestone One: Find Perfect Matchings

背景介绍：

你有一群人都在上同一门课。这门课让人们在作业上结对工作，而人们都想找人一起工作。每个人都在相互交谈，他们各自列出了一个他们愿意与之合作的人的名单。然后问题出现了——鉴于每个人的偏好，是否有可能将每个人配对，使他们与他们想一起工作的人结成伙伴？

我们可以通过为每个人画一个圆圈，然后画线连接那些愿意彼此合作的人，来形象地说明这个问题。下面是一个例子，说明这可能是什么样子：

我们可以用Map<string, Set<string>>来表示上面的情形：

“A”: { “B” }, “B”: { “A”, “C”, “E”, “G” }, “C”: { “B”, “D”, “G” }, “D”: { “C”, “G”, “H” }, “E”: { “B”, “F” }, “F”: { “E”, “G” }, “G”: { “B”, “C”, “D”, “F”, “H” }, “H”: { “D”, “G” }

把想一起工作的人组成了伙伴，并以一种方式使每个人都能匹配到伙伴。这样的安排被称为perfect matching。

我们的任务是编写函数：

bool hasPerfectMatching(const Map<string, Set<string>>& possibleLinks, Set<Pair>& matching)

该函数输入可能的连接，寻找是否存在满足上面要求的perfect matching，如果不存在则返回false，如果存在的话则返回true，并且集合matching中储存这一perfect matching

注意到Pair类型，这是一个储存着两个字符串的类型，相关的使用如下：

Pair greek1("Castor", "Pollux"); // Create a pair.
Pair greek2("Pollux", "Castor"); // Same pair; order doesn't matter.
if (greek1 == greek2) {
// true; elements are the same even though in a different order
}
cout << greek1.first() << ", "
<< greek1.second() << endl; // Prints Castor, Pollux

老师同样给了提示：

  • 在每一个点，你会发现要么每个人都已经配对了，要么每个人还没有配对
  • 如果是第一种情况的话，那么我们已经结束了，即找到了perfect matching
  • 如果是第二种情况的话，选择一个没有配对的人，尝试所有将其与未配对的人配对的可能方式

函数具体实现如下：

bool hasPerfectMatching(const Map<string, Set<string>>& possibleLinks, Set<Pair>& matching) {
    if (possibleLinks.isEmpty()){
        return true;
    }
    else{
        string choice = possibleLinks.firstKey();
        if (possibleLinks[choice].isEmpty()){
            return false;
        }
        else{
            for(string partner : possibleLinks[choice]){
                Map<string, Set<string>> remainLinks = possibleLinks;
                remainLinks.remove(choice);
                remainLinks.remove(partner);
                Pair newmatch(choice, partner);
                for(string keyleft : remainLinks){
                    if(remainLinks[keyleft].contains(choice))
                        remainLinks[keyleft] -= choice;
                    if(remainLinks[keyleft].contains(partner))
                        remainLinks[keyleft] -= partner;
                }
                if (hasPerfectMatching(remainLinks, matching)){
                    matching.add(newmatch);
                    return true;
                }
            }
        }
    }
    return false;
}

讲一个我一开始犯的小错，在最后递归调用的时候，一开始没有将增加新的match放入if语句之下，这就导致了前面就算是错误的组合也会被加入集合中。还是需要仔细考虑啊orz

Milestone Two: Find Maximum-Weight Matchings

背景介绍：

前面介绍的perfect match在现实生活中并不是最优的：

  • 无法配对奇数个学生
  • 无法描述不同配对之间的区别

进一步细化模型我们引入权重来说明：

虽然是奇数个，但是我们总能找到权重最大的组合，这种方式被称为maximum-weight matching

我们的任务是编写函数：

Set<Pair> maximumWeightMatching(const Map<string, Map<string, int>>& possibleLinks)

输入带有权重的可能链接，返回最大权重的组合。

这段代码我已经不记得写了几天了hhh，所以还是非常有难度的一段代码，但是自己写完真的非常的爽！

老师在作业中讲到，不要照着Milestone One中的代码去改，一开始我还是觉得这两个问题非常像，所以照着上一问的代码修改了好久，但是在反复阅读材料之后感觉这不是一个问题，推倒重来才写成功了。

首先采用了wrapping function的方式，定义了递归部分的函数maxmatching：

Set<Pair> maximumWeightMatching(const Map<string, Map<string, int>>& possibleLinks) {
    Set<Pair> matching;
    return maxmatching(possibleLinks, matching);
}

之后的思路是：

  • 对于传入的一个网络，先随便选择一个节点，我们称之为choice
  • 这个节点有两种情况，要么被选中与其他人结对，要么自己独立不被选中
  • 因此我们需要考虑isolated与linked两种情况，两种情况都要递归
  • 递归中传递网络的时候记得**“修剪”**网格！

完整代码如下：

int weightcount(const Map<string, Map<string, int>>& possibleLinks, Set<Pair> & matching){
    int result = 0;
    if (!matching.isEmpty()){
        for (Pair matches: matching){
            result += possibleLinks[matches.first()][matches.second()];
        }
    }
    return result;
}


Set<Pair> maxmatching(const Map<string, Map<string, int>> Links, Set<Pair> matching){

    /* Empty graph*/
    if (Links.isEmpty()){
        return matching;
    }
    /* choose one node */
    string  choice = Links.firstKey();

    /* store all possible value and matches */
    Map<Set<Pair>, int> MaxWeightPair;

    /* isolated case: leave this node alone */
    // modify the rest of graph
    Map<string, Map<string, int>> RemainLinks = Links;
    RemainLinks.remove(choice);
    for (string allnode: Links){
        if (Links[allnode].containsKey(choice)){
            RemainLinks[allnode].remove(choice);
        }
    }
    // Recursion!
    Set<Pair> isolatematch = maxmatching(RemainLinks, matching);
    // store this value
    MaxWeightPair[isolatematch] = weightcount(Links, isolatematch);



    /* Pair case: select one linked node and calculate its value */
    for (string node: Links[choice]){
        // avoid negtive links
        if (Links[choice][node] < 0)
            continue;
        // create and try this pair
        Pair newmatch = {choice, node};

        // continue to modify the rest of graph
        Map<string, Map<string, int>> RemainLinks = Links;
        RemainLinks.remove(choice);
        RemainLinks.remove(node);
        for (string allnode: Links){
            if (Links[allnode].containsKey(node)){
                RemainLinks[allnode].remove(node);
            }
            if (Links[allnode].containsKey(choice)){
                RemainLinks[allnode].remove(choice);
            }
        }
        // recursion!
        Set<Pair> thismatch = maxmatching(RemainLinks, matching + newmatch) + matching;
        MaxWeightPair[thismatch] = weightcount(Links, thismatch);
    }

    /* return match of max value */
    matching = MaxWeightPair.firstKey();
    for (Set<Pair> allmatch :MaxWeightPair){
        if(MaxWeightPair[allmatch] > MaxWeightPair[matching]){
            matching = allmatch;
        }
    }
    return matching;
}

Set<Pair> maximumWeightMatching(const Map<string, Map<string, int>>& possibleLinks) {
    Set<Pair> matching;
    return maxmatching(possibleLinks, matching);
}

Part Three: Disaster Planning

问题的背景是灾害物品储备。由于储备物资十分的昂贵，因此只维持少数几个城市储备物资是合理的，这样做的挑战是要弄清楚把资源放在哪里，以便：

  1. 我们不会花太多的钱储备超过我们的需要
  2. 我们不会让任何城市离应急物资太远

我们定义一个disaster-ready状态，它是指，要么一个城市有着应急物资储备，要么它紧挨着有应急物资储备的城市。

我们的任务是编写一个函数：

bool canBeMadeDisasterReady(const Map<string, Set<string>>& roadNetwork,
                            int numCities,
                            Set<string>& supplyLocations)

输入如下图所示的路网，最大的物资储备城市数，判断是否能够在该区域不超过最大允许物资储备城市数的前提下，达到disaster-ready状态。并且如果可行的话，函数应该将那些物资储备城市储存在集合supplyLocations中。

注意到这个问题可以近似于一个有着数量限制的组合问题，一个偷懒的办法便是直接列出最大组合数下的所有组合，但是当上限增加时，复杂度将会非常大。（想一想$C_m^n$的形式就可以知道）

老师为我们提供了以下的思路：

对于任意一个城市，证明它能够达到disaster-ready的形式的尝试有且仅有以下两种：

  • 选择这个城市，作为物资储备的城市
  • 选择这个城市紧挨的城市之一，作为物资储备的城市

如果这个选择能够递归地覆盖所有城市的话，那么我们就实现了我们最终的目标。但是如果未能实现的话，我们跳至下一个选项。

注意对于负的上限需要使用error函数报错。

一开始我直接莽着写了全部列举的方法……结果还是败在了组合数增长的速度上，最后又采取了wrapping function的方法，传递一个中间参数uncoveredCity，即每次不需要从头再选择一遍，从还没选的里面选就行。

可能有人会和我一开始一样，怀疑这种方法怎么解决两个相邻的物资储备城市的情况？

但其实，这种情况是可以被考虑到的，在其他点位的邻居的遍历中，就会考虑到这个情况。（所以还是要纸笔推一推……）

完整代码如下：

bool DisasterHelper(const Map<string, Set<string>>& roadNetwork,
                    int numCities,
                    Set<string>& supplyLocations,
                    Set<string>& uncoveredCity) {
    /* if all cities have been covered ,return ture. */
    if (uncoveredCity.isEmpty()){
        return true;
    }
    else if (numCities == 0 && !uncoveredCity.isEmpty()){
        return false;
    }
    /* An empty net always stays ready */
    if (roadNetwork.isEmpty()){
        return true;
    }

    string city = uncoveredCity.first();

    /* choose the city as supplylocation */
    supplyLocations.add(city);
    Set<string> trythis = uncoveredCity - city - roadNetwork[city];
    if(DisasterHelper(roadNetwork, numCities - 1, supplyLocations,trythis)){
        return true;
    }
    else{
        supplyLocations.remove(city);
    }

    /* choose surrounding cities as supplylocation */
    for (string neighbor: roadNetwork[city]){
        supplyLocations.add(neighbor);
        trythis = uncoveredCity - neighbor - roadNetwork[neighbor];
        if(DisasterHelper(roadNetwork, numCities - 1, supplyLocations,trythis)){
            return true;
        }
        else{
            supplyLocations.remove(neighbor);
        }
    }
    return false;
}


bool canBeMadeDisasterReady(const Map<string, Set<string>>& roadNetwork,
                            int numCities,
                            Set<string>& supplyLocations) {
    if (numCities < 0){
        error("Haiya!Negtive number!");
    }
    Set<string> uncoveredCity;
    for (string city: roadNetwork){
        uncoveredCity += city;
    }
    return DisasterHelper(roadNetwork, numCities, supplyLocations, uncoveredCity);
}
