Stream Ⅱ

cout & cin

一共有四种标准的iostreams

  • cin：标准输入
  • cout：标准输出（缓冲区
  • cerr：标准错误流（未缓冲
  • clog：标准错误流（缓冲

cin和iss的区别在于，如果缓冲区不存在东西时，会等用户输入东西

三个为什么带有>>的cin是一场噩梦的理由：

  1. cin读取一整行到缓冲区中，但是返回的是根据空白符分割的片段
  2. 缓冲区中的其他信息会使得cin可能不会等待用户输入
  3. 当前面的cin失效后，后面的cin也会失效

所以如果我们想读取一整行时，可以采用getline(cin,var)

但是仍然会出现一些不方便的bug，当我们想将所有的东西复制到一个不依赖于外部的stream中，我们使用stringstream!

getInteger

int getInteger()
{
    while (true)
    {
        string line;
        int result;
        char trash;
        if (!getline(cin,line)) throw domain_error(...);
        istringstream iss(line);
        if (iss >> result && !(iss >> trash)) return result;
        iss.clear();
        iss.ignore(numeric_limits<streamsize>::max(),'\n');
    }
}

下面我们来解析这个读取整数的函数getInteger()

 if (!getline(cin,line)) throw domain_error(...);

这一行代码尝试去读取一整行，因为缓冲区此时是空的，所以他会提示用户输入

istringstream iss(line);

使用刚刚读取的一行line，创建一个istringstream

if (iss >> result && !(iss >> trash)) return result;

这一行先在iss中尝试读取int的result，之后再尝试读取任何的trash（注意上节课讲到的iss的读取位置的变化，第二次读取trash时会自动寻找）

如果if判定失败，则清空流并且再次尝试，因为出现了一些垃圾

反之则返回我们想要的result

实际上，由于每一次都在创建不同的流，所以最后两行的清空操作可以省略

Summary

stringstream并不是用来彻底取代stream的，其仍然有自身的适用范围

  • 在处理字符串时
    • 化简“/./a/b”为“/a”
  • 规范输入输出时
  • 分析不同的类型时
    • 例如getinteger()
