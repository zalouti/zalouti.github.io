Part One: The Sierpinski Triangle

第一题的背景是自相似分形的著名例子——Sierpinski三角形。

  • $0$阶的Sierpinski三角形是一个实心的正常三角形
  • 一个$n$阶的Sierpinski三角形，由三个$n-1$阶的Sierpinski三角形组成，每个都是主三角形的$\frac{1}{4}$大，排列在一起，使它们角对角相接。

我们的任务是编写一个函数：

void drawSierpinskiTriangle(GWindow& window,
                            double x0, double y0,
                            double x1, double y1,
                            double x2, double y2,
                            int order)

这个函数输入待绘制三角形的图窗window，以及三个三角形的点与三角形的阶数，然后绘制该阶数的Sierpinski三角形。如果阶数order小于0的话，需要使用error函数报错。

绘制三角形的话，需要使用下面已经提供的函数：

void drawTriangle(GWindow& window,
                  double x0, double y0,
                  double x1, double y1,
                  double x2, double y2) {
    window.setColor("black");
    window.fillPolygon({ x0, y0, x1, y1, x2, y2 });
}

这个函数输入待绘制三角形的图窗window，以及三个三角形的点，绘制一个黑色的实心三角形。

实现思路：

  • 对于负数阶的三角形，报错
  • 对于0阶的三角形，使用drawTriangle绘制一个三角形
  • 对于n > 0 阶的三角形，利用递归绘制三个**次级(order - 1)**的三角形，注意输入的顶点即可

下面是drawSierpinskiTriangle函数实现：

void drawSierpinskiTriangle(GWindow& window,
                            double x0, double y0,
                            double x1, double y1,
                            double x2, double y2,
                            int order) {
    if (order < 0)
        error("Haiya! Order should be positive!");
    else if (order == 0)
        drawTriangle(window,x0,y0,x1,y1,x2,y2);
    else
    {
        drawSierpinskiTriangle(window,x0,y0,(x0+x1)/2,(y0+y1)/2,(x0+x2)/2,(y0+y2)/2,order - 1);
        drawSierpinskiTriangle(window,x1,y1,(x0+x1)/2,(y0+y1)/2,(x1+x2)/2,(y1+y2)/2,order - 1);
        drawSierpinskiTriangle(window,x2,y2,(x0+x2)/2,(y0+y2)/2,(x1+x2)/2,(y1+y2)/2,order - 1);
    }
}

Part Two: Human Pyramids

背景是人塔问题，我们的目标是计算$n$阶的人塔中某个人所承受的重量。

Milestone 1: The Initial Implementation

由受力分析不难得知每个人所承受的重量，我们的任务是编写函数：

double weightOnBackOf(int row, int col, int pyramidHeight)

要求为：

  • 假设每个人自身体重均为160磅
  • 函数必须递归地编写
  • 当坐标(row, col)不在人塔中时报错，坐标定义如下图，不难发现列编号永远小于等于行编号

#include "error.h" //Remember to include this line!
double weightOnBackOf(int row, int col, int pyramidHeight) {
    if(col > row || col < 0 || row < 0|| row > pyramidHeight)
        error("Haiya! Wrong cooridinates!");
    else if (col == 0 && row ==0)
        return 0.0;
    else if (col == 0 && row !=0)
        return 80.0 + 0.5 * weightOnBackOf(row - 1, col ,pyramidHeight);
    else if (col == row && row != 0)
        return 80.0 + 0.5 * weightOnBackOf(row - 1, col - 1 ,pyramidHeight);
    return 80.0 * 2 + 0.5 * weightOnBackOf(row - 1, col - 1,pyramidHeight) + 0.5 * weightOnBackOf(row - 1, col ,pyramidHeight);
}

设计思路是分类讨论递归：

  • 对于坐标(0,0)直接返回0
  • 对于最左侧的，返回上层人体重和所被负重量总和的一半，编号为row - 1, col
  • 对于最右侧的，返回上层人体重和所被负重量总和的一半，编号为row - 1, col - 1
  • 对于中间的，返回上层人体重和所被负重量总和的一半，编号为row - 1, col - 1与row - 1, col

Milestone 2: Speeding Things Up

但是上面的递归方式其实是非常低效的，一个原因便是由于人塔中蕴含的一个菱形：

我们会发现，在计算坐标(3,1)与(3,2)时，需要调用坐标(2,1)两次，当阶数递增时这会非常浪费。因此，一个简单的方法便是采用一个容器储存中间的步骤。

我们的任务是：

  • 将原来的简单的递归代码，利用包装函数wrapper function与某一容器，提高程序运行速度
  • 伪代码如下
Ret functionRec(Arg a, Table& table) {
    if (base-case-holds) {
        return base-case-value;
    } else if (table contains a) {
        return table[a];
    } else {
        do-some-work;
        table[a] = recursive-step-value;
        return recursive-step-value;
    }
}
Ret function(Arg a) {
    Table table;
    return functionRec(a, table);
}

此处我采用的容器是Map<GridLocation, double>，GridLocation用来储存坐标位置，对应一个double类型的重量，此处需要注意的是，在负责递归部分函数中，需要递归调用的**不是包装函数！**不然的话我们的容器无法储存下来，总是在不断地初始化，这样就达不到想要的效果了，代码如下：

#include "grid.h"
#include "map.h"
double weightOnBackOfHelper(int row, int col, int pyramidHeight, Map<GridLocation, double> & weightKnown){
    GridLocation coordinate ;
    coordinate.col = col;
    coordinate.row = row;
    if(col > row || col < 0 || row < 0|| row > pyramidHeight)
        error("Haiya! Wrong cooridinates!");
    if(col == 0 && row ==0)
        return 0.0;
    else if(weightKnown.containsKey(coordinate))
        return weightKnown[coordinate];
    else{
        if (col == 0 && row !=0)
            weightKnown[coordinate] = 80.0 + 0.5 * weightOnBackOfHelper(row - 1, col ,pyramidHeight, weightKnown);
        else if (col == row && row != 0)
            weightKnown[coordinate] = 80.0 + 0.5 * weightOnBackOfHelper(row - 1, col - 1 ,pyramidHeight, weightKnown);
        else
            weightKnown[coordinate] = 80.0 * 2
                    + 0.5 * weightOnBackOfHelper(row - 1, col - 1 ,pyramidHeight, weightKnown)
                    + 0.5 * weightOnBackOfHelper(row - 1, col ,pyramidHeight, weightKnown);
        return weightKnown[coordinate];
    }
}

double weightOnBackOf(int row, int col, int pyramidHeight) {
    Map<GridLocation, double> weightKnown;
    return weightOnBackOfHelper(row, col, pyramidHeight, weightKnown);
}

Part Three: What Are YOU Doing?

题目的背景是，在英语口语中，强调不同的单词会表达出不同的意思，我们的任务是编写一个函数：

Set<string> allEmphasesOf(const string& sentence);

输入一个包含一句话的string，返回一个包含所有强调方式的Set<string>，需要注意的是allEmphasesOf输出应该包含原有的标点符号。

但是如何分割单词呢？老师已经贴心地提供了函数：

Vector<string> tokenize(const string& sentence);

这个函数输入一个包含一句话的string，返回这句话的断词之后的Vector，例如：

输入：Quoth the raven, “Nevermore.”

tokenize输出：[Quoth] [ ] [the] [ ] [raven] [,] [ ] ["] [Nevermore] [.] ["]

按照上课讲的方法，我们写出递归的步骤：

  • Decision at each step:
    • 我们是否还有可以选择的单词
  • Options at each decision:
    • 将这个单词大写
    • 将这个单词小写
  • Information we should store along the way:
    • 我们已经强调过的单词组合
    • 剩下需要考虑的单词

因此我们发现，原函数给定的参数const string不方便递归，因此我们选择采用包装函数，传入的参数为已经考虑过的部分与未考虑的部分：

Set<string> allEmphasesOfHelper(Vector<string> decisionmade, Vector<string> decisiontomake)

Set<string> allEmphasesOf(const string& sentence) {
    Vector<string> tokens = tokenize(sentence);
    Vector<string> init;
    return allEmphasesOfHelper(init, tokens);
}

首先考虑最基本的情况：

  • 当传入这句话是一个空字符串时，直接返回一个空集合
  • 拓展一下，当我们没有可以选择的单词时，直接返回我们创造的集合

这对应了allEmphasesOfHelper的第一部分：

Set<string> allEmphasesOfHelper(Vector<string> decisionmade, Vector<string> decisiontomake){
    Set<string> emphasis;
    if (decisiontomake.isEmpty())
        return emphasis;

接着考虑递归部分，首先我们需要判断Vector中的元素是否是单词（因为后面的大小写只能应用于字母），作业中提示了，所给的文本只需要判断首字母即可，自然想到可以运用函数isalpha()

  • 对于字符部分，首先需要删除decisiontomake中的元素，接着将其全部大写或者小写加入decisionmade中，此处我才用了临时的tokencopy储存对应的单词。

    • 上面操作结束之后，我们的状态decisionmade与decisiontomake发生了变化，此处自然想到应该调用递归函数allEmphasesOfHelper
    • 当Vector中最后一个元素被加入decisionmade时，将这种组合加入我们输出的集合emphasis中，否则会出现集合中只有部分句子的情况

  • 对于非字符部分，唯一的区别便是不需要大小写转化，因此并不需要调用递归函数

完整代码实现如下：

#include "WhatAreYouDoing.h"
#include "strlib.h"
#include <cctype>
using namespace std;

string Vec2Str(Vector<string> tokens){
    string result = "";
    for (string str:tokens){
        result += str;
    }
    return result;
}

Set<string> allEmphasesOfHelper(Vector<string> decisionmade, Vector<string> decisiontomake){
    Set<string> emphasis;
    if (decisiontomake.isEmpty())
        return emphasis;
    else{
        Vector<string> tokencopy = decisiontomake;
        for(int i = 0; i < tokencopy.size(); i++){

            if(isalpha(decisiontomake[0][0])){
                // include lowercase-version of this word
                decisiontomake.remove(0);
                decisionmade += toLowerCase(tokencopy[i]);
                if(decisiontomake.isEmpty())
                    emphasis.add(Vec2Str(decisionmade) + Vec2Str(decisiontomake));
                emphasis += allEmphasesOfHelper(decisionmade, decisiontomake);

                // include uppercase-version of this word
                decisionmade.remove(decisionmade.size() - 1);
                decisionmade += toUpperCase(tokencopy[i]);
                if(decisiontomake.isEmpty())
                    emphasis.add(Vec2Str(decisionmade) + Vec2Str(decisiontomake));
                emphasis += allEmphasesOfHelper(decisionmade, decisiontomake);
            }
            else{
                decisiontomake.remove(0);
                decisionmade += tokencopy[i];
                if(decisiontomake.isEmpty())
                    emphasis.add(Vec2Str(decisionmade) + Vec2Str(decisiontomake));}
        }
    }
    return emphasis;
}

Set<string> allEmphasesOf(const string& sentence) {
    Vector<string> tokens = tokenize(sentence);
    Vector<string> init;
    return allEmphasesOfHelper(init, tokens);
}

Part Four: Shift Scheduling

这道题目的背景是，作为一个资本家，如何安排换班可以实现剥削员工最大的剩余价值（雾

对于每一班，我们已知如果某名员工工作的话所产生的价值，安排员工换班需要遵守两条规则：

  1. 每个员工每周有固定的工作总时长
  2. 每个员工不能连续工作两班

我们的任务是编写一个函数：

Set<Shift> highestValueScheduleFor(const Set<Shift>& shifts, int maxHours)

输入一个各班次的集合与员工工作的最长时间，返回一个产生最大价值的班次安排集合。Shift是一个定义在Shift.h中的struct，包含关于班次的时间信息（它是一周中的哪一天，什么时候开始，什么时候结束）。可能还会需要用到Shift.h中以下的函数：

int lengthOf(const Shift& shift); // Returns the length of a shift.
int valueOf(const Shift& shift); // Returns the value of this shift.
bool overlapsWith(const Shift& one, // Returns whether two shifts overlap.
const Shift& two);

老师详细地讲解了这个递归策略，让我直接机翻一段：

  • 无论怎么递归，首先需要从轮班中选择一些班次
    • 如果员工不能参加的话，要么是因为与已有班次重叠，要么是因为超过了每周的总时间，那么唯一的选择便是不给员工这个班次
    • 否则我们有两个选择
      • 让该员工参加这个班次
      • 让员工不参加这个班次
    • 这两个选项中可能某一个更好或者二者相等。在我们尝试之前，并不知道哪一个更好，所以（递归）探索这两个选项，并报告哪一个能带来更多的价值

其实在看到作业中写任务四的时候建议先完成任务三，所以我认为这是一个疯狂暗示，因此采用第三题的方法，先采用一个包装函数：

Set<Shift> highestValueScheduleForHelper(const Set<Shift> & curShifts, 
                                         const Set<Shift> & remainShifts,
                                         int maxHours);
 
Set<Shift> highestValueScheduleFor(const Set<Shift>& shifts, int maxHours) {
    Set<Shift> curShifts;
    Set<Shift> remainShifts = shifts;
    if(maxHours < 0)
        error("Haiya! Negtive maxhours!");
    else if(maxHours == 0)
        return curShifts;
    else
        return highestValueScheduleForHelper(curShifts, remainShifts, maxHours);
}

实现思路：

  • 如果现在没有可以选择的排班，这就意味着递归结束，返回现在的排班
  • 首先我们从待选择的集合中拿出一个元素，这就意味着之后所有递归的待选择状态减一（因为我们已经考虑过这个元素了）
  • 对于限制条件，采用常见的flag即可
  • 如果满足条件，则分两种情况讨论
    • 加入这个元素，现在的排班集和增加了这个元素，调用递归并计算其价值
    • 不加入这个元素，直接返回现在的排班状态，调用递归并计算其价值
    • 比较价值得出返回哪一个集合
  • 如果不满足条件，直接返回递归的待选择状态减一的集合，相当于忽略这个元素

完整代码实现如下：

int valueOfAllShift(const Set<Shift>& curShifts){
    int result = 0;
    for(Shift shift:curShifts){
        result += valueOf(shift);
    }
    return result;
}

int timeOfAllShift(const Set<Shift>& curShifts){
    int time = 0;
    for(Shift shift:curShifts){
        time += lengthOf(shift);
    }
    return time;
}

Set<Shift> highestValueScheduleForHelper(const Set<Shift> & curShifts, 
                                         const Set<Shift> & remainShifts,
                                         int maxHours){
    Set<Shift> shiftwith, shiftwithout;
    if(remainShifts.isEmpty())
        return curShifts;
    else{
        int flag = 1;
        Shift elem = remainShifts.first();
        Set<Shift> remaincopy = remainShifts;
        remaincopy = remaincopy - elem;
        for (Shift shift: curShifts){
            if (overlapsWith(elem, shift))
            {
                flag = 0;
            }
        }
        if( timeOfAllShift(curShifts) + lengthOf(elem) > maxHours){
            flag = 0;
        }
        if (flag){
            /* not include this element */
            shiftwithout = highestValueScheduleForHelper(curShifts, remaincopy, maxHours);
            int valuewithout = valueOfAllShift(shiftwithout);

            /* include this element */

            shiftwith = highestValueScheduleForHelper(curShifts + elem, remaincopy, maxHours);
            int valuewithin = valueOfAllShift(shiftwith);

            /* choose highest value */
            if (valuewithin >= valuewithout)
                return shiftwith;
            else
                return shiftwithout;
        }
        else
            return highestValueScheduleForHelper(curShifts, remaincopy, maxHours);
    }
}
